# RSS News Aggregator API

A comprehensive Node.js API service that aggregates and filters financial news from multiple RSS feeds, including market data, earnings reports, analyst ratings, and SEC filings. The service provides curated, real-time financial news relevant to major stock exchanges (NYSE, NASDAQ, OTCQB, TSXV, OTCQX).

## üöÄ Features

- **Multi-Source RSS Aggregation**: Combines news from Business Wire, Globe Newswire, PR Newswire, AccessWire, Wall Street Journal, Barron's, and Reuters
- **Smart Filtering**: Filters news based on market exchanges, market cap thresholds, and configurable regex patterns
- **AI-Powered Content Filtering** (Optional): Uses AI (via Bytez SDK) to automatically filter out class action lawsuits and legal investigation news when enabled
- **Earnings Calendar**: Tracks upcoming earnings releases for the next 2 weeks
- **SEC Filings Integration**: Retrieves recent SEC filings for companies with upcoming earnings
- **Analyst Ratings**: Aggregates analyst ratings and price target updates
- **Market Gainers/Losers**: Provides feeds for top market gainers and losers
- **Caching System**: External caching via MockAPI to reduce redundant data fetching
- **Real-time Updates**: Automatically fetches and updates feeds within specified time windows
- **Downloadable Archive**: Provides ability to download all collected data as a ZIP file

## üõ† Installation

### Prerequisites

- Node.js 20.11.1 or higher
- npm or yarn package manager

### Local Setup

```bash
# Clone the repository
git clone https://github.com/rakeshbade/combine-rss-as-api.git
cd combine-rss-as-api

# Install dependencies
npm install

# Start the server
npm start
```

The server will start on port 3000 by default (or the PORT environment variable if set).

## ‚öôÔ∏è Configuration

### Environment Variables

- `PORT` - Server port (default: 3000)
- `TZ` - Timezone (default: "America/New_York")
- `NODE_ENV` - Environment mode (default: "production")
- `ENABLE_AI_FILTER` - Enable AI-based content filtering (default: false, set to 'true' to enable)
- `BYTEZ_API_KEY` - API key for Bytez AI service (required when ENABLE_AI_FILTER is true)

### AI Filtering (Optional)

AI filtering can be controlled in two ways:

**1. Per-Request (Query Parameter):**
Add `aiFilter=true` to any request to enable AI filtering for that specific request:
```bash
curl "http://localhost:3000/rss?blog=allwires&aiFilter=true"
curl "http://localhost:3000/feed-all?aiFilter=true"
```

**2. Globally (Environment Variable):**
Set environment variables to enable AI filtering for all requests:
```bash
export ENABLE_AI_FILTER=true
export BYTEZ_API_KEY=your_api_key_here
```

When enabled, the service uses AI (Bytez SDK with OpenAI GPT-OSS-20B model) to automatically detect and filter out:
- Class action lawsuits
- Legal investigations
- Securities fraud cases
- Shareholder litigation announcements
- Law firm investor alerts
- Legal deadline reminders

The AI filter works asynchronously after regex filtering and caches results to minimize API calls.

**Note:** AI filtering is disabled by default and requires `BYTEZ_API_KEY` to be set. If the API key is not configured, AI filtering will be disabled even if requested.

### RSS Feed Configuration

Feed configurations are located in the `config/` directory:

- **allwires.json** - News wire services (Business Wire, Globe Newswire, PR Newswire, AccessWire)
- **barrons.json** - Barron's market news
- **wsj.json** - Wall Street Journal deals section
- **reuters.json** - Reuters US market news
- **learn.json** - Educational/learning content
- **ratings.json** - Analyst ratings sources
- **regex.js** - Filtering patterns and exclusion rules

### Market Data Configuration

The service filters news based on companies with market cap > $1 billion across major exchanges:
- NYSE
- NASDAQ
- OTCQB
- TSXV
- OTCQX

## üîå API Endpoints

### GET `/rss?blog={source}`

Retrieve RSS feed for a specific source.

**Parameters:**
- `blog` (required) - Source identifier (allwires, barrons, wsj, reuters, learn)
- `aiFilter` (optional) - Set to "true" to enable AI filtering for this request (default: false)

**Example:**
```bash
curl "http://localhost:3000/rss?blog=allwires"

# With AI filtering enabled
curl "http://localhost:3000/rss?blog=allwires&aiFilter=true"
```

**Response:** XML RSS feed

---

### GET `/feed-all`

Retrieve aggregated feed from all configured sources.

**Parameters:**
- `forceReload` (optional) - Set to "false" to use cached data (default: "true"). When true, fetches fresh data before returning response.
- `exclude` (optional) - Comma-separated list of sources to exclude (e.g., "learn,ratings")
- `aiFilter` (optional) - Set to "true" to enable AI filtering for this request (default: false)

**Example:**
```bash
curl "http://localhost:3000/feed-all"

# Use cached data (faster response)
curl "http://localhost:3000/feed-all?forceReload=false"

# Exclude specific sources
curl "http://localhost:3000/feed-all?exclude=learn,ratings"

# With AI filtering enabled
curl "http://localhost:3000/feed-all?aiFilter=true"

# Combined
curl "http://localhost:3000/feed-all?exclude=learn&aiFilter=true&forceReload=true"
```

**Response:** Combined XML RSS feed

---

### GET `/sec-earnings`

Retrieve SEC filings for companies with upcoming earnings releases.

**Example:**
```bash
curl "http://localhost:3000/sec-earnings"
```

**Response:** XML RSS feed of SEC filings

---

### GET `/ratings`

Get analyst ratings for companies with upcoming earnings.

**Parameters:**
- `ignoreCompanies` (optional) - Companies to exclude from ratings

**Example:**
```bash
curl "http://localhost:3000/ratings"
```

**Response:** XML RSS feed of analyst ratings

---

### GET `/gainers`

Retrieve top market gainers feed.

**Example:**
```bash
curl "http://localhost:3000/gainers"
```

**Response:** XML RSS feed

---

### GET `/losers`

Retrieve top market losers feed.

**Example:**
```bash
curl "http://localhost:3000/losers"
```

**Response:** XML RSS feed

---

### GET `/update-market`

Update market data (companies and market caps).

**Parameters:**
- `forceReload` (optional) - Force refresh of market data

**Example:**
```bash
curl "http://localhost:3000/update-market?forceReload=true"
```

**Response:** JSON array of market data

---

### GET `/download`

Download all collected data as a ZIP archive.

**Example:**
```bash
curl "http://localhost:3000/download" -o data.zip
```

**Response:** ZIP file containing all data files

### Key Components

**Fetch Service** (`services/fetch.js`)
- Supports multiple fetching strategies: Axios, Puppeteer, custom scrapers
- Parses RSS/XML feeds and normalizes data
- Filters posts based on time windows, regex patterns, and market cap
- Applies AI filtering to remove legal/class action content

**AI Filter Service** (`services/aiFilter.js`)
- Uses Bytez SDK with GPT-OSS-20B model
- Detects class action lawsuits and legal investigations
- Implements caching to reduce API calls
- Extensible for additional content classification needs

**Feed Utilities** (`utils/feed.js`)
- Generates RSS feeds from JSON data
- Manages external cache via MockAPI
- Detects and filters duplicate entries

**Earnings Service** (`services/earnings.js`)
- Fetches earnings calendar from TradingView
- Retrieves SEC filings for companies with upcoming earnings
- Caches data with 12-hour TTL

**Market Service** (`services/market.js`)
- Updates market data from Financial Modeling Prep API
- Filters companies by market cap threshold ($1B+)
- Stores symbol and market cap data

## üîß Development

### Running Locally

```bash
# Install dependencies
npm install

# Start development server
npm start
```

### Adding New RSS Sources

1. Create a configuration file in `config/` (e.g., `newsource.json`)
2. Add the source to `config/index.js`
3. (Optional) Add filtering regex in `config/regex.js`
4. The source will be automatically available via `/rss?blog=newsource`

### Filtering Rules

Edit `config/regex.js` to customize filtering:

```javascript
module.exports = {
  sourcename: {
    include: /pattern-to-include/i,
    exclude: /pattern-to-exclude/i
  }
};
```

### Web Scraping Configuration

For sources requiring web scraping (like Barron's or AccessWire), configure in the JSON file:

```json
{
  "blogUrl": "https://example.com",
  "container": "article",
  "header": ".title",
  "description": ".summary",
  "link": "a.link",
  "date": ".publish-date",
  "loadType": "puppeteer"
}
```
